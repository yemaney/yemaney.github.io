Data Structures
  • Lists

Lists

Lists are ordered mutable sequences of multiple object references. The objects can be heterogeneous. A simple list can be created by wrapping comma separated values in square bracket for example my_list = [1, 2.03, "hello", None].

Use case

Since lists are mutable sequences, they are used to create collections of objects which will be either changed or iterated over.

Dictionaries

Dictionaries are collections of key: value pairs, with the condition that each key is unique. A simple dictionary can be created by put comma separated key:value pairs in curly brackets for example my_dict = {"breakfast": "cereal", "lunch": "soup", "dinner": "steak"}

Use case

The main benefit of dictionaries over lists is their insertion and search efficiency. So these are used when data is expected to be frequently accessed.

Tuples

Tuples are comma separated values enclosed in round brackets.my_tuple = (1, 2, 3)

Use case

Unlike lists they are immutable. So they are usually used to store values that are not expected to change.

If Else

Used to control the execution of code, buy making sure certain conditions are met in order for them to run.


x = secret_letter 
if  x == a: # first condition
	print('a')
elif x == b: # other conditions can be indicated using elif (else if)
    print('b')
else: # this is used if  all other conditions are not met
    print('c')

For Loops

Used to execute code for each element within an iterable.


count = 0
for i in range(10): # for every integer between 0 and 9 inclusively
	count += 1 # increment count by 1

While Loops

Used to execute code as long as some condition holds True


count = 0
while count < 10:
	count += 1 # increment count by 1 as long as it is less than 10

Break, Pass, Continue

These are used to further control the loops from above. breakis used to exit a loop early. continueis used to jump to the next step in the loop. passis used to signal that nothing is done.


count = 0
while count < 10:
	count += 1 # increment count by 1 as long as it is less than 10
	
	# exit the while loop early if count is an even integer
	if count % 2 == 0:
		break

Try, Except, Finally

These are used in order to handle errors. 


my_list = [1, 2, "three"]
for element in my_list:
	try:
		print(element / 2)
	except TypeError: # if element is not a number print a warning
		print('element is not a number')

Functions, Decorators, and Closures

Functions

A function can be understood as a sequence of related operation that are used to perform some task and optionally return a value. In python function are defined using the def keyword. The arguments for the function are defined in the parenthesis (). Keyword arguments are used to define defaults


# define simple arithmetic function
def add_to_two(x, y=2):
	"""add x and y and return the result"""
	return x + y

# using the function
sum1 = add_to_two(1) # y is not passed because it has a default value
print(sum)
# 3

sum2 = add_to_two(1, y=5) # y is passed, overriding the default value
print(sum)
# 6

Decorators

Functions are first class objects in python, so they can be used as arguments themselves to other functions. Decorators take advantage of this. Decorators are functions design to take another function as an argument in order to augment its behavior. Decorators can be used on other functions using the @decorator syntax over the function you want to alter.

The common pattern in decorator functions is:

  1. Create an outer function that takes a function funcas an argument
  2. Create an inner function inside the outer function
  3. The inner function executes func along with its custom code then returns the results
  4. The outer function finally returns the inner function

import time
from typing import Callable
# implementing a decorator that prints how long an function execution takes

def timer(func): # define outer function that takes a function (func) as an argument
	def timed(*args, **kwargs): 	# define inner function that can take any arguments        
		t0 = time.perf_counter() 	# start counter
		result = func(*args) 	# perform original function (func)
		elapsed = time.perf_counter() - t0 	# calculate elapsed time

		print(elapsed)
		return result
	return timed

@timer
add_to_two(1) # using the timer decorator on the function from earlier

Idiomatic Python and Best Practices

Python is a very versatile language that often offers many ways of performing the same task. But there is usually a preferred option, that balances readability and efficiency. This is said to be idiomatic python or pythonic

Don't loop using index


arr = [1,2,3,4]

# naive approach
for i in range(len(arr)):
	print(arr[i])

# pythonic approach (less clutter)
for num in arr:
	print(num)

# if you still need the index for some other reason
for i, num in enumerate(arr):
	print(i)
	print(num)


# in case of looping through a dictionary
obj = {"one": 1, "two": 2, "three": 3}

for key, value in obj.items():
	print(key)
	print(value)

use built in function when you can


# calculate sum of array
arr = [1,2,3,4]

# naive approach
total = 0
for num in arr:
	total += num

# using built in sum function
total = sum(arr)
